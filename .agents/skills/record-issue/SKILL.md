# Record Issue — 问题归档

## 触发条件

用户说以下任意一句时激活：
- "记录这个问题"
- "记一下这个 bug"
- "把这个问题归档"
- "记录一下"
- "mark this issue"

## 流程

### Step 1: 确认问题背景

从当前对话上下文中提取：
- **问题现象**：发生了什么、报错信息
- **排查过程**：尝试了哪些方向
- **根因**：最终定位到的原因
- **解决方案**：怎么修复的
- **关键代码**：修复前后的代码片段（如有）

如果上下文信息不足，反问用户补充。

### Step 2: 确定归档目录

根据问题所属领域选择 `docs/` 下的目录：

| 领域 | 目录 | 示例 |
|------|------|------|
| 构建工具（Rspack/SWC/Vite/webpack） | `docs/构建工具/` | 端口占用、DefinePlugin 配置 |
| 路由 / 鉴权（React Router / Guard） | `docs/鉴权/` | 401 未跳转、Token 过期 |
| 状态管理（MobX） | `docs/状态库/` | observer 未包裹、state 不响应 |
| 数据库（Prisma/SQLite） | `docs/数据库/` | 迁移失败、适配器配置 |
| 服务端（NestJS/Hono/Express） | `docs/后端/` | DI 注入失败、500 Internal |
| 样式 / UI（Tailwind/CSS） | `docs/样式/` | 白屏、样式不生效 |
| 通用 / 其他 | `docs/工程化/` | 环境变量、monorepo 结构 |

目录不存在则创建。

### Step 3: 生成 MD 文件

文件名：`{问题关键词}.md`，如 `NestJS-DI注入失败.md`、`前端-白屏-process-is-not-defined.md`

文件内容格式：

```markdown
# {标题}

> **日期**：{YYYY-MM-DD}  
> **标签**：{NestJS, DI, tsconfig}

## 现象

{描述问题的表现，附上报错信息}

## 排查

{尝试过的方法、排除过的方向}

## 根因

{一句话总结最终原因}

## 解决

{具体的修复步骤，附关键代码}

## 预防

{如何在未来避免同类问题}
```

### Step 4: 确认

告诉用户文件已生成的位置（`docs/xxx/yyy.md`）。

## 反模式

- ❌ 不记录在根目录，必须放在 docs 子目录下
- ❌ 只有现象没有解决方案时不归档（等解决了再记）
- ❌ 文件名用中文，不用拼音缩写
